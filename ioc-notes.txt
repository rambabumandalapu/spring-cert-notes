Overall explanation
destroy

@Deprecated
public void destroy()
Deprecated. as of Spring Framework 5.0, in favor of close()

close

public void close()

Close this application context, destroying all beans in its bean factory.

Delegates to doClose() for the actual closing procedure. Also removes a JVM shutdown hook, if registered, as it's not needed anymore.

registerShutdownHook

public void registerShutdownHook()

Register a shutdown hook named SpringContextShutdownHook with the JVM runtime, closing this context on JVM shutdown unless it has already been closed at that time.

Delegates to doClose() for the actual closing procedure.

### 🔄 Quick Walkthrough
1. **Instantiation** → Bean is created (constructor).  
2. **Populate Properties** → Dependencies injected (setter/constructor).  
3. **Aware Interfaces** → Bean learns its name/context (`BeanNameAware`, `ApplicationContextAware`).  
4. **BeanPostProcessor (before init)** → Pre-initialization tweaks.  
5. **Initialization** → Runs `@PostConstruct`, `afterPropertiesSet()`, or custom init method.  
6. **BeanPostProcessor (after init)** → Post-initialization tweaks.  
7. **Destruction** → Runs `@PreDestroy`, `DisposableBean.destroy()`, or custom destroy method.
